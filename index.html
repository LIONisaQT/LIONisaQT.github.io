Game Design and Development Projects

Shroommates

Made with Unreal Engine (currently work in progress)

Shroommates is third-person stealth game where users play as a sentient mushroom trying to grow in secret in a UCSC student's apartment. The mushroom can grow by consuming food and staying out of direct sunlight. Players must adapt to different sizes, as size dynamically alters speed and hiding ability. Complete the objectives while trying to keep out of sight of the AI human! Check out the Shroommates repository on GitHub.

  • Lead developer: Leading and delegating tasks to six other programmers.
  • AI programmer: Quickly implemented working temporary AI sufficient for prototyping purposes within a few days. Currently porting old AI functions and behaviors into newer, more robust AI, who contains an improved behavior tree.
  • Generalist programmer: Implementing game behaviors as needed in various areas.
  • Point person for communication and technical QA with professors

MinShmup

Written in Java using libGDX and Android Studio (currently on hiatus due to school)

MinShmup is a simple, minimalistic shoot-em up for Android. The intention of the project is for me to learn in-depth about and play with various methods and functions libGDX has to offer through a simple game. I also wanted to make a simple game feel amazing to play, taking inspriration from grapefrukt games' juicy-breakout. For the player, I want to make a game that looks aethetically pleasing, satisfying to play, and easy to pick up and play whenever they have some time to kill. As a developer, I want to make a light-weight game with robust, easy-to-read code, as well as use the project to exercise good game development practices. MinShmup currently doesn't have a public repository on GitHub, but will in the upcoming weeks.

  • Feels great to play, even in its prototype stage – the player receives pleasing and appropriate visual and audio feedback for various in-game actions.
  • Fully takes advantage of libGDX's object pooling system to handle disposable objects, thus preventing any memory leaks and decreasing overall memory usage.
  • More libGDX methods to be implemented to optimize the game even further in the future.

Beat 'Em Up!

Prototyped with Unity

A rhythm-based fighting game prototype. Play as fighters from various genres of music in a musical beatdown. Players are incentivized to hit on the beat to deal more damage, rather than hitting off-beat and dealing minimal damage. This game was three-week project for a senior game design studio class, just like Shroommates. At the end of the quarter, the team voted to move on with Shroommates and abandon Beat 'Em Up! Check out the repository here.

  • Project manager: Leading and delegating tasks to a team of programmers and artists.
  • Programmed base character class: Implemented player attacks and added animations from images provided by artists.
