MinShmup |

Made with libGDX (currently on hiatus due to school) (solo project)

"Game juice." -Tad Leckman

MinShmup is a simple, minimalistic shoot-em up for Android. The intention of the project is for me to learn in-depth about and play with various methods and functions libGDX has to offer through a simple game. I also want to make a simple game feel amazing to play, taking inspriration from grapefrukt games' juicy-breakout. My goal as a designer is to reimagine a simple game that feels amazing to play. MinShmup currently doesn't have a public repository on GitHub, but will in the upcoming weeks.

  • Programming engaging visual feedback for some in-game actions through a camera shake manager.
    • Getting hit and destroying enemies adds “trauma”, which the manager uses to scale both rotational and translational shake in a natural and impactful way.
  • Fully takes advantage of libGDX's object pooling system to handle disposable objects, thus preventing any memory leaks and decreasing overall memory usage.
  • More libGDX methods to be implemented to optimize and juice up the game even further in the future.

Battlegrounds |

the Board Game (Systems Designer)

"Where are we dropping, boys?" -procrastinating student at 2AM

A board game based off of the recent popular trend of battle royale games and a systems design exercise. You've seen the thousands of PUBG and Fortnite knock-offs for PC and smartphones, now check out the board game! Manage a team of three units as you try to win that chicken dinner! Each unit has up to ten actions points to use to move around the map, open chests for loot, and shoot other players. Last team standing wins! View the presentation here.

  • Balanced reaction point system, which helps units getting shot at by granting them immediately spendable action points to prevent immediate death.
  • Balanced damage, range, accuracy (determined with die roll), and usage cost for 4 different weapons.
  • Programmed interactive game core loop using Machinations. Download the XML here, and open it within Machinations.
  • Transferred ideas, rules, and mechanics to visually clean documents for presentation.

Food Now! |

A MERN stack application (Project Owner and back-end developer)

"Ew no I ate there a couple of days ago, choose another place." -person who no longer has friends

Food Now! is a web application that aims to solve the problem of indecisive friends choosing a place to eat by being the final say in picking restaurants. The app takes in everyone's food preferences, a distance and price maximum, and then returns a random restaurant that respects the distance and price limit while best fulfilling at least one of those preferences. Uses React for front-end design, Express for the server framework, and Google Firebase for user login. Check out the repository here, and make sure to leave a review!

  • Led and delegated tasks to five other developers.
  • Sent user parameters from front-end to Yelp, handled list of restaurants returned by Yelp, and passed a random restaurant back to front-end to display to the user.

Gotcho Back |

HACK UCSC 2016: Tech Cares category, 1st place (Developer)

"Oh no, not Gotcho Back!" -your potentional kidnapper

According to multiple studies such as this one, a good amount of women feel it is unsafe for them to walk outside at night by themselves. Enter Gotcho Back, a safety app prototype for Android. Users pick a contact and set a timer to get home. If the timer runs out, the phone will vibrate and ask the user if they need more time. If the user does not respond within a specific timeframe, the phone will send a text to the contact and ring. After further lack of response, the app will call the contact. Feel safe at the repository here. Click the Devpost link for win proof.
